STAFFD pricing + generation direction, ratified with SA 2026-06-24 (numbers still red-pen-able, but the STRUCTURE is locked — do not re-litigate):

**Pricing philosophy** — price on the VALUE delivered (the team/the work), NOT on compute consumed. Anchor = "an on-call team for less than one hour of a freelancer's rate" (ceiling = the labor it defers: first hire $3-5k/mo, VA, agency retainer; floor = NOT "$20 ChatGPT" framing). As model costs fall ~10x/yr, margin widens automatically instead of forcing repricing — that's the future-proofing.

**Bury the meter.** The per-click credit decision ("Confirm — 8 credits", "you have 6000 video credits") IS the fear we're eliminating ("prove the average person doesn't need to fear AI"). Model quality becomes a PLAN BENEFIT, never a per-click charge. Fast stuff (images ~$0.015, fast video ~$0.60) is unlimited-feeling (fair use); only cinematic (~$2.50 veo3/sora) gets a real monthly allowance.

**Limits rule (SA's, locked):** caps apply ONLY to cinematic. NEVER interrupt a generation mid-project — always finish what's started. Gate only at the START of the NEXT project ("used your N cinematic videos, resets in X days — or grab a Cinema extension"). Monthly allowance that resets, not a punishing cooldown timer. Extension/Cinema packs = the opt-in escape valve.

**Upsell vs à la carte:** newcomers reaching for premium → UPSELL the plan (desire moment). Per-click à la carte for a scared Starter = a UX-law violation (re-introduces the meter). À la carte is acceptable ONLY as power-user overage on Pro/Agency past their included allowance.

**Packaging:** base subscription (the team/departments) + capability packs (Cinema-Studio, tiered, for the creator persona) + extension packs (overage). Onboarding should capture primary intent ("what will you use your team for most?") to personalize AND aim the upsell (video/meme creator → Cinema-Studio pack, not 9 departments).

**Plan table (committed W95.9, 46f30ab — red-pen anytime, one-line config in app/api/_lib/billing/plan-benefits.ts):** Starter $39 / Growth $79 / Pro $149 / Agency $450 (prices HELD — add value, don't reprice). Allowances counted in CLIPS (each generated ~4s shot = one generation; ~8 shots = one stitched 30s commercial; commercialsFromClips() for copy). Everyday video (veo3-fast ~$0.60) = INVISIBLE fair-use ceiling 25/50/100/250 per plan (bounded so a power user can't run cost away, but the 95% never see it). Cinematic (veo3/sora ~$2.50) = the one VISIBLE allowance: 0/8/24/60 per plan (Agency tops at 60≈7 commercials — SA red-penned 120→60 and capped everyday). Images unlimited (~$0.015). Designed on AVERAGE usage (~25-30% of ceiling → ~80% gross margin); rare ceiling-hitter is the best upgrade/pack candidate. Cinema extension packs +10/$39, +30/$99. Free trial MUST include a cinematic so they see the ceiling.

**Video assembly = STAFFD auto-stitches, on purpose (ratified direction).** A cinematic "video" is multiple shots STITCHED into the finished commercial — the staffer delivers the cut, not loose clips. So #3 (multi-shot stitch) is REQUIRED, not optional, for cinematic to mean anything. PLUS a lightweight editor (reorder shots, regenerate ONE weak shot, trim, captions, music) — agencies live in it, everyone else takes the auto-cut. Stitching can't run on Vercel (ffmpeg too heavy for serverless) → runs on a video worker; recommended v1 path = a cloud assembly API (Shotstack/Creatomate-style, JSON clip-list → rendered mp4, ~$0.30/render, invisible per Model B3 [[project_staffd_model_b3]]), move to self-hosted ffmpeg on Railway later if volume justifies. The #2 allowance counter counts cinematic CLIPS (per muapi premium call) and is independent of the stitch, so #2 engine and #3 stitch are separable.

**Generation models (shipped W95.7.3e-vid2, 9ebf186):** NEVER bet the product on one model — keep the swappable registry in routing.ts and update picks as the leaderboard turns. Current picks (OpenAPI-verified, costs from SA's live muapi dashboard): video quick=seedance-2 / veo3-fast, pro=veo3-fast ($0.60), premium=veo3 / sora-2 ($2.50); image quick=flux-2, pro=flux-2-pro/imagen4, premium=nano-banana-pro/midjourney. Key fact: pixverse-v6 billed $0.585 for garbage while veo3-fast bills $0.60 for usable output — the old models were worse AND not cheaper. Credit charge is by TIER not model, so model swaps are pricing-neutral. Do NOT fork/mirror the muapi GitHub app templates — a fork makes the identical POST /api/v1/<slug> call STAFFD already makes (zero benefit, fragments the app); call muapi directly off the verified OpenAPI. **ARCHITECTURE TRUTH (re-litigated 3×, settle it): image+video pixels are ALWAYS rendered by muapi; our "agents" are LLM text-intelligence (Anthropic/Groq) that write the PROMPT — there is no in-house draw/render model and there can't be one without a diffusion/video model = muapi. So we need muapi for BOTH image and video. A good output = a good prompt + a good render + inline display (June 3 logo proved it). The recurring quality bugs are OUR prompt-construction + OUR rendering, never the plumbing.** W95.9.x fixed two such regressions: (1) CommandCenter rendered the result as a raw markdown/text link → now inline <img>/<video> + Download (DepartmentRoom was already correct); (2) the IMAGE enricher was still the maximalist "preserve every word/render all text" art-director → fed a creative BRIEF it rendered a photo of the brief ("Apex Solutions") → now DISTILLS to the actual artifact, logo-aware, never renders strategy-doc text. Next lever if logos still aren't logo-grade: the Logo Designer SPECIALIST should write a vivid image prompt directly (June-3 style) instead of a strategy brief + visual-style learning ([[project_staffd_roadmap_gaps]]).
